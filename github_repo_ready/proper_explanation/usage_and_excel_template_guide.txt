Maple Guild Participation OCR Excel - 사용방법 및 엑셀 템플릿 안내
=================================================================

대상 파일
- 기준 코드: final_batch_queue_koreanv5_fgsplit_hybridmatch_fillrules.py

목적
- 메이플스토리 길드 참여 현황 스크린샷 여러 장을 한 번에 등록한다.
- 공식 길드원 명단을 불러온다.
- 닉네임 crop, 수로 점수(F열), 플래그 점수(G열)를 추출한다.
- 결과를 엑셀로 저장해 사람이 검토/수정하기 쉽게 만든다.

핵심 운영 방향
- 완전 자동 OCR보다 반자동 정리 툴에 가깝다.
- 닉네임은 crop 이미지를 보고 사람이 직접 적어도 된다.
- 수로 점수와 플래그 점수는 최대한 자동으로 정확하게 추출한다.


1. 실행 전 준비
----------------
필수 패키지 예시
- python 3.10 이상 권장
- PySide6
- paddleocr
- opencv-python
- numpy
- requests
- beautifulsoup4
- lxml
- openpyxl
- pillow

예시 설치
pip install PySide6 paddleocr opencv-python numpy requests beautifulsoup4 lxml openpyxl pillow

실행 예시
python final_batch_queue_koreanv5_fgsplit_hybridmatch_fillrules.py


2. 프로그램 사용 순서
---------------------
1) 공식 길드원 명단 불러오기
- 월드를 선택한다.
- 길드 이름을 입력한다.
- [공식 길드원 불러오기] 버튼을 누른다.
- 공식 홈페이지에서 길드원 명단을 수집한다.

2) 원본 이미지 여러 장 등록
- [원본 이미지 여러 장 추가] 버튼으로 스크린샷을 한 번에 여러 장 넣는다.
- 기본 row 수를 먼저 정한 뒤 추가할 수 있다.
- 추가 후 표에서 이미지별 row 수를 직접 수정할 수 있다.
- 잘못 넣은 이미지는 선택 삭제 또는 전체 비우기가 가능하다.

3) OCR 실행
- [목록 전체 OCR 실행] 버튼을 누른다.
- 등록된 이미지들을 순서대로 처리한다.
- 각 이미지마다 열 자동 검출, row segmentation, F/G 판독, 닉네임 처리 등이 수행된다.

4) 자동 보정
- [공식 명단으로 자동보정] 버튼을 눌러 공식 길드원 명단 기준으로 보정할 수 있다.
- 최신 버전은 하이브리드 매칭 방식을 사용한다.
  - 유사도 0.75 이상: 고신뢰로 간주
  - 유사도 0.75 미만: 중복 허용 가능
  - 유사도 0.35 미만: 결과는 유지하되 실제 닉네임 셀을 회색 표시

5) 엑셀 저장
- [엑셀 저장] 버튼을 누른다.
- 기본 파일명은 "길드이름_ocr_result.xlsx" 형식으로 제안된다.
- 저장 후 엑셀에서 실제 닉네임을 수기로 입력/수정하면 된다.


3. 입력 이미지 권장 조건
------------------------
- 한 장의 스크린샷 안에 row 간격이 일정할수록 좋다.
- 열 위치가 크게 흔들리지 않는 캡처가 좋다.
- 너무 심한 압축 이미지보다 원본에 가까운 PNG가 유리하다.
- 같은 UI 배율, 비슷한 해상도일수록 안정적이다.
- row 수는 직접 알고 있는 값을 넣는 방식이다.


4. 현재 코드 기준 처리 방식 요약
-------------------------------
- A열(닉네임): crop 이미지 생성 + OCR/보정 결과 저장
- F열(수로 점수): 숫자 판독 결과 저장
- G열(플래그 점수): 0 또는 1000 형태로 저장
- F/G는 엑셀 저장 시 쉼표를 제거한 숫자만 저장한다.
  - 예: 4,307 -> 4307
  - 예: 1,000 -> 1000


5. 결과 엑셀 템플릿
------------------
[시트 1] 결과
열 구조는 아래와 같다.

A열: 매칭 점수
- OCR 닉네임과 보정 닉네임 사이의 유사도 점수
- 색칠하지 않음

B열: OCR 닉네임
- OCR이 직접 읽은 원문 닉네임

C열: 보정 닉네임
- 공식 길드원 명단을 참고해 자동 보정된 닉네임

D열: 닉네임 이미지
- 해당 row에서 잘라낸 닉네임 crop 이미지
- 사람이 직접 보고 판별하기 위한 핵심 열

E열: 실제 닉네임
- 사용자가 직접 입력하는 수동 검토용 열
- 현재 버전에서 색상 규칙은 이 열에만 적용됨
  - 매칭 점수 >= 0.75 : 연한 보라색
  - 매칭 점수 < 0.35 : 연한 회색
  - 그 외: 색 없음

F열: 수로 점수
- 쉼표 없는 숫자만 저장
- 예: 7884, 16740, 0

G열: 플래그 점수
- 쉼표 없는 숫자만 저장
- 예: 0, 1000


6. 결과 엑셀 시트 구성
----------------------
시트명: 결과
- 실제 검토와 수정이 이루어지는 मुख्य 시트

시트명: 입력 정보
- 월드
- 길드 이름
- 공식 길드원 수
- 누적 OCR row 수
- 등록된 이미지 수
- 성공 처리 이미지 수

시트명: 처리 이미지 목록
- 파일명
- 전체 경로
- 입력 row 수
- 생성 row 수
- 상태

시트명: 공식 길드원 명단
- 공식 홈페이지에서 수집한 길드원 이름 목록


7. 색상 규칙
-----------
현재 버전 기준 색상은 "실제 닉네임(E열)"에만 적용된다.

- 연한 보라색: match_score >= 0.75
  -> 신뢰도가 높은 편이므로 빠르게 검토 가능

- 연한 회색: match_score < 0.35
  -> 저신뢰 결과이므로 사람이 우선 확인 필요

- 유사율 열(A열): 색칠하지 않음


8. 사용 팁
---------
- 닉네임은 이미지(D열)를 보고 사람이 적는 방식으로 운영해도 충분하다.
- 수로 점수와 플래그 점수는 자동 추출 결과를 우선 사용한다.
- 저신뢰 행(회색 E열)은 사람이 먼저 확인하면 빠르다.
- 보라색 E열은 비교적 신뢰도가 높은 편이지만, 최종 확인은 사람이 한다.
- 여러 장을 한 번에 넣되, 각 이미지 row 수는 꼭 맞게 입력하는 것이 좋다.


9. 현재 버전의 한계
------------------
- 닉네임 자동 보정은 여전히 오인식이 있을 수 있다.
- 다른 사용자 환경, 다른 UI 배율, 다른 캡처 품질에서는 성능이 달라질 수 있다.
- 닉네임 쪽은 완전 자동보다 반자동 검토 도구로 보는 것이 현실적이다.
- 처리 속도는 이미지 수와 row 수가 많아질수록 늘어난다.


10. 추천 운영 방식
-----------------
- 공식 길드원 명단 불러오기
- 이미지 여러 장 등록
- row 수 점검
- 목록 전체 OCR 실행
- 자동 보정 수행
- 엑셀 저장
- D열 닉네임 이미지를 보고 E열 실제 닉네임을 수기로 입력
- F/G 점수는 자동 추출 결과를 확인 후 필요 시만 수정


11. 한 줄 요약
-------------
이 도구는 "닉네임은 사람이 보고 적기 쉽게 crop 이미지와 참고 정보를 제공하고, 수로/플래그 점수는 빠르게 추출해 엑셀로 정리하는 반자동 툴"로 사용하는 것이 가장 적합하다.
